From package.json: `"node-cron": "^4.2.1"`. node-cron 4.x uses the same `timezone` option semantics as 3.x (verified in node-cron release notes). DST handling: wall-clock scheduling; when the configured hour:minute does not exist (spring-forward), the schedule skips that firing; when the hour:minute occurs twice (fall-back), node-cron fires once at the first occurrence. Combined with the DB UNIQUE(summary_date) constraint from Phase 20 and Phase 21's pre-flight SELECT, duplicate firings are idempotent no-ops.

"Yesterday" computation in timezone-aware form (matching Plan 22-01's Intl.DateTimeFormat pattern):
```typescript
function computeYesterday(now: Date, tz: string): Date {
  const fmt = new Intl.DateTimeFormat('en-CA', {
    timeZone: tz, year: 'numeric', month: '2-digit', day: '2-digit',
  });
  const todayLocal = fmt.format(now);           // e.g., '2026-04-19'
  const todayDate = new Date(todayLocal + 'T00:00:00Z');  // UTC midnight anchor
  return new Date(todayDate.getTime() - 86_400_000);      // yesterday UTC midnight
}
```
The cron fires at 23:00 local time; "yesterday" from that firing means the calendar day that just ended at midnight. We compute "the day before the current local date" — safe across DST because the `Intl.DateTimeFormat` computes the LOCAL date regardless of the wall-clock shift.

    ```typescript
    import { config } from '../config.js';
    import { logger } from '../utils/logger.js';
    import { runConsolidate } from './consolidate.js';

    /**
     * Compute the "yesterday" calendar date in config.proactiveTimezone.
     * Returns a Date anchored at UTC midnight for the prior calendar day in tz.
     *
     * Safe across DST because Intl.DateTimeFormat computes the LOCAL date
     * regardless of the wall-clock shift: on 2026-10-25 (fall-back in Paris)
     * the cron fires at 23:00 Paris time — the local date is 2026-10-25, and
     * yesterday is 2026-10-24 regardless of which of the two 23:00 instants
     * the timer uses.
     */
    function computeYesterday(now: Date, tz: string): Date {
      const fmt = new Intl.DateTimeFormat('en-CA', {
        timeZone: tz,
        year: 'numeric',
        month: '2-digit',
        day: '2-digit',
      });
      const todayLocal = fmt.format(now);             // 'YYYY-MM-DD' in tz
      const todayUtcMidnight = new Date(todayLocal + 'T00:00:00Z');
      return new Date(todayUtcMidnight.getTime() - 86_400_000);
    }

    /**
     * Cron-registered wrapper around runConsolidate. Computes yesterday's
     * calendar date in config.proactiveTimezone, invokes the consolidation
     * engine, and swallows any thrown error (CRON-01 — cron must never crash
     * the process; CONS-12 — failure notification is owned by runConsolidate).
     *
     * Exported for direct invocation by tests and by the backfill script
     * (OPS-01 in Phase 23) which calls runConsolidate directly rather than
     * through this wrapper.
     *
     * `now` is injectable for tests; defaults to wall-clock now.
     */
    export async function runConsolidateYesterday(now: Date = new Date()): Promise<void> {
      const yesterday = computeYesterday(now, config.proactiveTimezone);
      const yesterdayIso = yesterday.toISOString().slice(0, 10);

      logger.info(
        { yesterdayIso, timezone: config.proactiveTimezone },
        'episodic.cron.invoked',
      );

      try {
        await runConsolidate(yesterday);
      } catch (error) {
        logger.warn(
          {
            yesterdayIso,
            error: error instanceof Error ? error.message : String(error),
          },
          'episodic.cron.error',
        );
      }
    }
    ```
