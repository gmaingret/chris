1. **Normalize the date input.** Accept a Date; produce the local calendar date string in `config.proactiveTimezone` (format YYYY-MM-DD). This string is the value of `summary_date` in the inserted row AND the idempotency key for the pre-flight SELECT.

2. **Pre-flight idempotency SELECT (CONS-03 part 1).** Query `SELECT id FROM episodic_summaries WHERE summary_date = $1`. If a row exists, log at INFO level "episodic.consolidate.skip.existing" and return `{ skipped: 'existing' }`. Zero Sonnet calls.

3. **Entry-count gate (CONS-02).** Call `getPensieveEntriesForDay(date, config.proactiveTimezone)`. If the returned array has length 0, log at INFO "episodic.consolidate.skip.no-entries" and return `{ skipped: 'no-entries' }`. Zero Sonnet calls. Zero DB writes.

4. **Fetch auxiliary sources in parallel (CONS-08, CONS-09 data).**
   ```typescript
   const [contradictions, decisions] = await Promise.all([
     getContradictionsForDay(date, tz),
     getDecisionsForDay(date, tz),
   ]);
   ```

5. **Assemble the prompt (CONS-04, 05, 09, 10, 11 via Plan 21-02).**
   ```typescript
   const prompt = assembleConsolidationPrompt({
     summaryDate: localDateString,
     entries,
     contradictions,
     decisions,
   });
   ```

6. **Call Sonnet with structured output (CONS-01 core).** Use `messages.parse()` via the bumped SDK:
   ```typescript
   const response = await anthropic.messages.parse({
     model: SONNET_MODEL,
     max_tokens: 2000,
     system: [{ type: 'text', text: prompt, cache_control: { type: 'ephemeral' } }],
     messages: [{ role: 'user', content: 'Generate the episodic summary for this day.' }],
     response_format: zodResponseFormat(EpisodicSummarySonnetOutputSchema, 'episodic_summary'),
   });
   const parsed = response.parsed;  // typed, already validated by the SDK helper
   ```

   **Retry rule:** ONE retry on parse failure. Wrap the call in a small helper `callSonnetWithRetry(prompt)` that catches `ZodError` (from the SDK) and calls once more. Catch any other error (rate-limit, network, 5xx) and propagate to step 9. The second retry failure also propagates.

7. **Runtime importance floors (CONS-06 + CONS-07 belt-and-suspenders).**
   ```typescript
   let importance = parsed.importance;
   const hasRealDecision = decisions.some(
     (d) => (d.createdToday || d.resolvedToday) &&
       ['open', 'due', 'resolved', 'reviewed'].includes(d.lifecycleState),
   );
   if (hasRealDecision) importance = Math.max(importance, 6);
   if (contradictions.length > 0) importance = Math.max(importance, 7);
   ```

8. **Runtime Zod re-validation of the full insert row (CONS-01 belt-and-suspenders).**
   ```typescript
   const insertRow = parseEpisodicSummary({
     summary: parsed.summary,
     importance,
     topics: parsed.topics,
     emotional_arc: parsed.emotional_arc,
     key_quotes: parsed.key_quotes,
     summary_date: new Date(localDateString),  // coerce YYYY-MM-DD → Date
     source_entry_ids: entries.map((e) => e.id),
   });
   ```
   Any Zod error at this layer is a programmer error (SDK output was Sonnet-schema-valid but not insert-schema-valid) — throw and propagate to step 9.

9. **Insert with ON CONFLICT DO NOTHING (CONS-03 part 2 — belt-and-suspenders).**
   ```typescript
   const inserted = await db
     .insert(episodicSummaries)
     .values({ ...drizzleRowShape })
     .onConflictDoNothing({ target: episodicSummaries.summaryDate })
     .returning({ id: episodicSummaries.id });
   if (inserted.length === 0) {
     // Race: another caller inserted between step 2 and step 9. Log, return skipped.
     logger.info({ summaryDate: localDateString }, 'episodic.consolidate.skip.race');
     return { skipped: 'existing' };
   }
   return { inserted: true, id: inserted[0].id };
   ```

10. **Failure path (CONS-12).** Any error from steps 3–9 is caught at the top-level try/catch in runConsolidate. Action: `await notifyConsolidationError(date, err);` and `logger.error(...)`. Return `{ failed: true, error: err }`.

    ```typescript
    import { and, eq } from 'drizzle-orm';
    import { DateTime } from 'luxon';
    import { zodResponseFormat } from '@anthropic-ai/sdk/helpers/zod';
    import { anthropic, SONNET_MODEL } from '../llm/client.js';
    import { config } from '../config.js';
    import { db } from '../db/connection.js';
    import { episodicSummaries } from '../db/schema.js';
    import { logger } from '../utils/logger.js';
    import { assembleConsolidationPrompt } from './prompts.js';
    import {
      getPensieveEntriesForDay,
      getContradictionsForDay,
      getDecisionsForDay,
    } from './sources.js';
    import {
      EpisodicSummarySonnetOutputSchema,
      parseEpisodicSummary,
    } from './types.js';
    import { notifyConsolidationError } from './notify.js';

    export type ConsolidateResult =
      | { inserted: true; id: string }
      | { skipped: 'existing' | 'no-entries' }
      | { failed: true; error: unknown };

    const MAX_TOKENS = 2000;
    const REAL_DECISION_STATES = ['open', 'due', 'resolved', 'reviewed'];

    async function callSonnetWithRetry(prompt: string): Promise<
      ReturnType<typeof EpisodicSummarySonnetOutputSchema['parse']>
    > {
      const request = {
        model: SONNET_MODEL,
        max_tokens: MAX_TOKENS,
        system: [{ type: 'text' as const, text: prompt, cache_control: { type: 'ephemeral' as const } }],
        messages: [{ role: 'user' as const, content: 'Generate the episodic summary for this day.' }],
        response_format: zodResponseFormat(EpisodicSummarySonnetOutputSchema, 'episodic_summary'),
      };
      try {
        const response = await (anthropic.messages as any).parse(request);
        return response.parsed ?? EpisodicSummarySonnetOutputSchema.parse(response);
      } catch (firstErr) {
        logger.warn(
          { err: firstErr instanceof Error ? firstErr.message : String(firstErr) },
          'episodic.consolidate.sonnet.retry',
        );
        const response = await (anthropic.messages as any).parse(request);
        return response.parsed ?? EpisodicSummarySonnetOutputSchema.parse(response);
      }
    }

    /**
     * Generate the episodic summary for a single calendar date in config.proactiveTimezone.
     *
     * Requirements satisfied:
     * - CONS-01: orchestration wiring
     * - CONS-02: entry-count gate (pre-Sonnet)
     * - CONS-03: idempotency (pre-flight SELECT + ON CONFLICT DO NOTHING)
     * - CONS-06: decision-day importance floor (runtime clamp)
     * - CONS-07: contradiction-day importance floor (runtime clamp)
     * - CONS-12: failure notification via Telegram
     */
    export async function runConsolidate(date: Date): Promise<ConsolidateResult> {
      const tz = config.proactiveTimezone;
      const localDateStr = DateTime.fromJSDate(date, { zone: tz }).toISODate();
      if (!localDateStr) {
        const err = new Error(`Invalid date for consolidation: ${date.toISOString()}`);
        await notifyConsolidationError(date, err);
        return { failed: true, error: err };
      }

      try {
        // Step 2 — idempotency pre-flight
        const existing = await db
          .select({ id: episodicSummaries.id })
          .from(episodicSummaries)
          .where(eq(episodicSummaries.summaryDate, localDateStr))
          .limit(1);
        if (existing.length > 0) {
          logger.info({ summaryDate: localDateStr }, 'episodic.consolidate.skip.existing');
          return { skipped: 'existing' };
        }

        // Step 3 — entry-count gate
        const entries = await getPensieveEntriesForDay(date, tz);
        if (entries.length === 0) {
          logger.info({ summaryDate: localDateStr }, 'episodic.consolidate.skip.no-entries');
          return { skipped: 'no-entries' };
        }

        // Step 4 — fetch contradictions + decisions in parallel
        const [contradictions, decisions] = await Promise.all([
          getContradictionsForDay(date, tz),
          getDecisionsForDay(date, tz),
        ]);

        // Step 5 — assemble prompt
        const prompt = assembleConsolidationPrompt({
          summaryDate: localDateStr,
          entries,
          contradictions,
          decisions,
        });

        // Step 6 — Sonnet structured call with one retry
        const parsed = await callSonnetWithRetry(prompt);

        // Step 7 — runtime importance floors
        let importance = parsed.importance;
        const hasRealDecision = decisions.some(
          (d) =>
            (d.createdToday || d.resolvedToday) &&
            REAL_DECISION_STATES.includes(d.lifecycleState),
        );
        if (hasRealDecision) importance = Math.max(importance, 6);
        if (contradictions.length > 0) importance = Math.max(importance, 7);

        // Step 8 — Zod re-validate the full insert row
        const insertRow = parseEpisodicSummary({
          summary: parsed.summary,
          importance,
          topics: parsed.topics,
          emotional_arc: parsed.emotional_arc,
          key_quotes: parsed.key_quotes,
          summary_date: new Date(localDateStr + 'T00:00:00Z'),
          source_entry_ids: entries.map((e) => e.id),
        });

        // Step 9 — insert with ON CONFLICT DO NOTHING
        const drizzleRow = {
          summaryDate: localDateStr,
          summary: insertRow.summary,
          importance: insertRow.importance,
          topics: insertRow.topics,
          emotionalArc: insertRow.emotional_arc,
          keyQuotes: insertRow.key_quotes,
          sourceEntryIds: insertRow.source_entry_ids,
        };
        const inserted = await db
          .insert(episodicSummaries)
          .values(drizzleRow)
          .onConflictDoNothing({ target: episodicSummaries.summaryDate })
          .returning({ id: episodicSummaries.id });
        if (inserted.length === 0) {
          logger.info({ summaryDate: localDateStr }, 'episodic.consolidate.skip.race');
          return { skipped: 'existing' };
        }
        logger.info(
          { summaryDate: localDateStr, id: inserted[0].id, importance },
          'episodic.consolidate.complete',
        );
        return { inserted: true, id: inserted[0].id };
      } catch (err) {
        // Step 10 — failure path
        logger.error(
          { err: err instanceof Error ? err.message : String(err), summaryDate: localDateStr },
          'episodic.consolidate.error',
        );
        await notifyConsolidationError(date, err);
        return { failed: true, error: err };
      }
    }
    ```

    NOTE on `(anthropic.messages as any).parse`: the `messages.parse()` helper is GA in SDK ^0.90 but TypeScript types for it may vary by minor version. If the direct call type-checks, remove the `as any`. If it doesn't, keep the cast — the behavior is what the SDK docs promise regardless of local type inference.

    NOTE on the `cache_control` field: per research SUMMARY.md, prompt caching is a free optimization that saves ~90% on system tokens during backfill. Set it on the system block — zero cost if caching isn't hit.

    Use the exact column names from the Phase 20 schema (checked via grep against `src/db/schema.ts` AFTER Phase 20 has landed). If column naming differs (e.g., Phase 20 settled on `summary_date` column → `summaryDate` property), adjust the Drizzle select/insert accordingly.
