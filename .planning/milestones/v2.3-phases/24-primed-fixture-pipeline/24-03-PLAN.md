<objective>
Close SYNTH-03: run the REAL `runConsolidate()` engine against each synthetic day in a primed fixture and persist authentic Sonnet-generated episodic summaries back into the fixture's `episodic_summaries.jsonl`. Use **sibling-module composition** (Pattern 2 Option 3 — no production code mod, no monkey-patching of production modules, only pre-import singleton binding swap scoped to the script process) against a **throwaway Docker Postgres on port 5435** (D-04 port correction per RESEARCH §Pitfall 5 to avoid collision with `regen-snapshots.sh`'s port 5434).

Purpose: This plan enforces D-04's single most important design choice — *the engine we ship is the engine that produces our test data*. Any drift between fixture-generation consolidation and production consolidation becomes a latent bug source. By invoking `runConsolidate(date)` directly (not a mock, not a re-implementation), every production change to the consolidation engine is automatically reflected in future fixture regenerations.

**Placement choice (called out per orchestrator):** `scripts/synthesize-episodic.ts` ships as a **sibling module** (NOT an `--episodic` flag extension of `scripts/synthesize-delta.ts`). Rationale:
1. Plan 24-02's `synthesize-delta.ts` is already ~400 lines; adding a Docker lifecycle + migration apply + DATABASE_URL rewire inside it blows its context budget and muddies its single concern (Haiku + deterministic generators).
2. The two scripts have different failure modes: synth-delta fails on SSH/VCR/seed issues; synth-episodic fails on Docker/migration/port-5435 issues. Separating them makes operator triage clearer.
3. `scripts/regenerate-primed.ts` (Plan 24-04) composes both in sequence — operators who want the full pipeline call `regenerate-primed.ts`, not either sibling directly.

Output:
- `scripts/synthesize-episodic.ts` — throwaway PG5435 + sibling-composition `runConsolidate` loop + episodic_summaries JSONL dump
- `scripts/__tests__/synthesize-episodic.test.ts` — integration test asserting sibling-composition works end-to-end
</objective>

    // ── JSONL I/O ───────────────────────────────────────────────────────────

    async function loadJsonl<T>(path: string): Promise<T[]> {
      const exists = await stat(path).then(() => true).catch(() => false);
      if (!exists) return [];
      const rl = createInterface({ input: createReadStream(path, { encoding: 'utf8' }), crlfDelay: Infinity });
      const out: T[] = [];
      for await (const line of rl) {
        const trimmed = line.trim();
        if (trimmed) out.push(JSON.parse(trimmed) as T);
      }
      return out;
    }

    function bigintReplacer(_k: string, v: unknown): unknown {
      return typeof v === 'bigint' ? v.toString() : v;
    }

    async function writeJsonl<T>(path: string, rows: readonly T[]): Promise<void> {
      const tmp = `${path}.tmp-${process.pid}`;
      const out = createWriteStream(tmp, { encoding: 'utf8' });
      for (const row of rows) out.write(JSON.stringify(row, bigintReplacer) + '\n');
      await new Promise<void>((r) => out.end(r));
      // Atomic rename
      const { rename } = await import('node:fs/promises');
      await rename(tmp, path);
    }
