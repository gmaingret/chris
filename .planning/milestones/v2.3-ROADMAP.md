## Overview

A pre-M009 enabler milestone. Builds the organic+synthetic primed-fixture pipeline so every downstream milestone (M009–M014) can be validated immediately, without waiting real calendar time for episodic summaries to accumulate. Codifies a new project convention (D041) banning calendar-time data-accumulation gates in subsequent milestones.

The pipeline anchors test data in real prod (SSH to Proxmox 192.168.1.50, read-only pg_dump of 9 tables), extends it with deterministic synthetic days via Haiku style-transfer + real `runConsolidate()` episodic synthesis, and loads the result into the Docker Postgres test DB through `loadPrimedFixture(name)`. Auto-refreshes stale snapshots silently (24h TTL), is seed-reproducible (SHA-256 content-addressable VCR cache wraps Anthropic SDK), and ships with `scripts/regenerate-primed.ts` as a thin child-process composer.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions:**

- **v2.3 is standalone, not folded into M009** — reusable infrastructure across M009–M014 avoids per-milestone re-derivation of synthesis code.
- **Organic data scope = `source='telegram'` only** — matches M008.1 consolidation contract; synced sources (immich/gmail/gdrive) bloat fixtures without adding test signal.
- **Freshness policy = 24h auto-refresh** — silent invocation; `--no-refresh` opt-out for sandbox/air-gap; no warnings, no half-stale runs.
- **VCR hash key = SHA-256 over canonical-stringified request (keys sorted at every nesting level).** Sort-at-every-level is load-bearing — Anthropic SDK's nested `output_config.format` JSON-schema emitter does not preserve key order.
- **Deterministic UUID generator** — 4 × Mulberry32 draws → UUIDv4 layout — replaces `crypto.randomUUID()` at every synthesis site to close SYNTH-07.
- **Sibling-module composition (Pattern 2 Option 3)** is the entire no-production-code-mod contract for Plan 24-03.
- **Port 5435 locks the operator invocation lane** for `synthesize-episodic.ts` (disjoint from `regen-snapshots.sh`'s 5434).
- **`relational_memory` not "memories"** — REQ documents used the alias; actual Drizzle table is `relational_memory` (schema.ts:134). FK-cleanup uses real table name.
- **D041 codified** — *"no milestone may gate on real calendar time for data accumulation — use the primed-fixture pipeline instead."* In PLAN.md, CONVENTIONS.md, TESTING.md.
- **D-09 soft-fail default ships; `strictFreshness: true` option shipped for future use.** No in-tree consumer of strict today.
- **`primed-sanity.test.ts skip-guard`** — `describe.skip` when `tests/fixtures/primed/m008-14days/MANIFEST.json` is absent → `bash scripts/test.sh` stays green in sandbox/CI; flips to running once operator generates the fixture.

**Issues Resolved:**

- Pre-M009 dependency on real calendar time for episodic summary accumulation — eliminated via primed-fixture pipeline + D041 convention.
- VCR cache key correctness under nested JSON-schema emission — resolved via canonical sort at every nesting level.
- `synthesize-episodic.ts` port collision risk against `regen-snapshots.sh` — resolved via port 5435 (disjoint).
- FK-cleanup order alias confusion (`memories` vs `relational_memory`) — resolved via grep-enforced naming.
