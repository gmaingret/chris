**JSDoc block + DST-safe Luxon idiom** (verbatim shape from `sources.ts:74-83`):

```typescript
/**
 * Compute the next UTC fire instant for a ritual given current time + config.
 * DST-safe: uses Luxon's wall-clock arithmetic (`plus({ days })` operates on
 * local time, not UTC ms) so a daily ritual configured for 21:00 Paris fires
 * at 21:00 Paris on both 2026-03-29 (spring-forward, 23h day) and 2026-10-25
 * (fall-back, 25h day).
 *
 * Mirrors the canonical `dayBoundaryUtc` pattern in src/episodic/sources.ts:74-83.
 */
export function computeNextRunAt(now: Date, config: RitualConfig): Date {
  const tz = config.time_zone;
  const [hh, mm] = config.fire_at.split(':').map(Number);
  let target = DateTime.fromJSDate(now, { zone: tz })
    .startOf('day')
    .set({ hour: hh!, minute: mm!, second: 0, millisecond: 0 });
  if (target.toJSDate() <= now) {
    target = target.plus({ days: 1 });
  }
  // ... cadence switch ...
  return target.toUTC().toJSDate();
}
```

- All timezone math via Luxon `setZone` / `fromJSDate({ zone: tz })`. NEVER `setUTCHours` / `+86_400_000` (Pitfall 2/3 grep guard).

### Idempotency via atomic UPDATE...RETURNING with WHERE-clause guard

**Source:** `src/decisions/lifecycle.ts:75-137` (`transitionDecision` ŌĆö M007 D-28 precedent)
**Apply to:** `src/rituals/idempotency.ts` (or `scheduler.ts`)

**Pattern:**
```typescript
const updated = await db.update(<table>)
  .set({ /* new state */ })
  .where(and(
    eq(<table>.id, id),
    /* concurrency guard: only succeed if our observed prior state is still authoritative */
  ))
  .returning();
if (updated.length === 0) {
  // race lost
}
```

**Why:** Postgres row-level locking + WHERE-clause predicate gives atomic check-and-set in one round-trip ŌĆö no SELECT-then-UPDATE race window.

### DST-safe wall-clock advancement via Luxon

**Source:** `src/episodic/sources.ts:74-83` (`dayBoundaryUtc`)
**Apply to:** `src/rituals/cadence.ts` (`computeNextRunAt`)

**Pattern:**

**Why:** Luxon's `plus({ days })` operates on local wall-clock, NOT UTC ms. On a 23h spring-forward day, `+ 1 day` in local time advances by 23h UTC; on a 25h fall-back day, by 25h UTC.
