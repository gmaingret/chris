## Conventions specific to Phase 29

- **Pure-function discipline (`weekly-review-prompt.ts` + `weekly-review-sources.ts`):** No DB calls in the assembler. No fs reads. The assembler returns a string; the sources module returns plain JS objects. Tests run in microseconds.
- **CONSTITUTIONAL_PREAMBLE explicit injection** (Pitfall 17 mitigation): Both the assembler (`weekly-review-prompt.ts`) AND the assertion test grep (`grep -L 'CONSTITUTIONAL_PREAMBLE' src/rituals/weekly-review-prompt.ts` returns zero hits) — defense in depth.
- **v3/v4 dual Zod schema at SDK boundary** — same shape as `src/episodic/consolidate.ts:33-81`. v3 is the contract; v4 is for `zodOutputFormat`. Both kept in lock-step.
- **Retry cap = 2** (mirrors `callSonnetWithRetry` + extends to fallback after cap). Logged as `chris.weekly-review.fallback-fired`.
- **Tag override at `storePensieveEntry` boundary** — explicit `epistemic_tag: 'RITUAL_RESPONSE'` parameter bypasses Haiku auto-tagger. Cross-phase coordination with Phase 26 (whichever ships first defines the API).
- **Per-phase migration files** — Phase 29 = 0009. Append-only, INSERT-only seed. ON CONFLICT (name) DO NOTHING for idempotency.
- **Live test scaffolding (Plan 29-04)** — file ships with `skipIf(!process.env.ANTHROPIC_API_KEY)`. Phase 30 owns execution + excluded-suite update.
- **D031 header rendering at Telegram-send time** — NOT in the prompt. The prompt asks Sonnet for structured `{observation, question}`; the `'Observation (interpretation, not fact):'` header is a UX-layer constant prepended at fire time.
