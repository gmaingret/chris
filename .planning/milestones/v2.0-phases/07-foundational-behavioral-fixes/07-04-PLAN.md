    const chatKey = chatId.toString();
    const refusalResult = detectRefusal(text);
    if (refusalResult.isRefusal) {
      // Detect language first for acknowledgment language
      const previousLanguage = getLastUserLanguage(chatKey);
      const refusalLanguage = detectLanguage(text, previousLanguage);
      setLastUserLanguage(chatKey, refusalLanguage);

      addDeclinedTopic(chatKey, refusalResult.topic, refusalResult.originalSentence);
      const ack = generateRefusalAcknowledgment(refusalLanguage);
      await saveMessage(chatId, 'USER', text, 'JOURNAL');

| T-07-06 | Tampering | Declined topic injection | mitigate | Topics are stored as plain strings in the DeclinedTopicEntry struct. | T-07-07 | Information Disclosure | Language state leakage | accept | Single-user system (D009). Session language Map is keyed by chatId but only one user exists. No cross-user leakage possible. |
| T-07-08 | Spoofing | Refusal bypass | accept | Refusals are best-effort UX improvement, not a security boundary. If a user circumvents their own refusal patterns, there's no harm — they're the only user. |
</threat_model>
