  <action>
    Create `src/scripts/audit-pensieve.ts` with these exported functions and types:

    1. **Types:**
    ```
    export type AuditStatus = 'correct' | 'incorrect' | 'unrelated';
    export type AuditAction = 'kept' | 'soft_deleted' | 'would_correct';
    export interface AuditResult {
      entryId: string;
      content: string;
      status: AuditStatus;
      action: AuditAction;
      groundTruthKey?: string;
      correctedContent?: string;
    }
    ```

    2. **`matchEntryToGroundTruth(content: string)`** — Examines entry content against GROUND_TRUTH entries using keyword matching per D-04 scope:
    - For each ground-truth entry, check if the content mentions relevant keywords for that fact category
    - Rental property: if content mentions 'Cagnes-sur-Mer' in context of rental/apartment/rented AND ground truth says 'Golfe-Juan' -> incorrect
    - Move direction: if content mentions moving 'from Georgia to Saint Petersburg' or similar inverted direction -> incorrect
    - Birth date/place, nationality, FI target, business entities: check for factual accuracy against GROUND_TRUTH_MAP values
    - Returns `{ matched: boolean; key?: string; isCorrect?: boolean; issue?: string }`

    3. **`generateCorrectedContent(key: string, originalContent: string)`** — Generates a corrected content string by replacing the incorrect fact with the correct value from GROUND_TRUTH_MAP. Return format: `"[Audit correction] {correct fact statement}"`.

    4. **`formatAuditReport(results: AuditResult[], isDryRun: boolean)`** — Generates markdown string per D-05 with:
    - Header: `# Pensieve Audit Report`
    - Date/time of run
    - Summary: total reviewed, correct, incorrect, unrelated, actions taken
    - Table with columns: Entry ID (first 8 chars), Content (first 80 chars), Status, Action, Ground Truth Key
    - Footer: dry-run or wet-run indicator

    5. **`auditPensieve(options: { dryRun: boolean; reportPath: string })`** — Main audit function:
    - Queries all entries where `epistemicTag IN ('FACT', 'RELATIONSHIP') AND deletedAt IS NULL` using Drizzle `db.select()` with `isNull(pensieveEntries.deletedAt)` and `inArray(pensieveEntries.epistemicTag, ['FACT', 'RELATIONSHIP'])`
    - For each entry, calls `matchEntryToGroundTruth(entry.content)`
    - If matched AND incorrect AND NOT dryRun:
      - Soft-delete: `await db.update(pensieveEntries).set({ deletedAt: new Date() }).where(eq(pensieveEntries.id, entry.id))`
      - Generate corrected content via `generateCorrectedContent()`
      - Insert corrected entry: `const [corrected] = await db.insert(pensieveEntries).values({ content: correctedContent, epistemicTag: 'FACT', source: 'audit', metadata: { auditCorrectedFrom: entry.id, groundTruthKey: key } }).returning()`
      - Synchronous embedding per D-02: `await embedAndStore(corrected.id, correctedContent)`
      - Record action as 'soft_deleted'
    - If matched AND incorrect AND dryRun: record action as 'would_correct'
    - If matched AND correct: record action as 'kept'
    - If not matched: record as 'unrelated', action 'kept'
    - Call `formatAuditReport(results, options.dryRun)` and write to `options.reportPath` using `fs.writeFileSync`
    - Return `{ results, mutationCount }` where mutationCount is number of actual soft-deletes performed

    6. **`main()`** — CLI entry point:
    - Parse `--dry-run` flag from `process.argv`
    - Default report path: `.planning/phases/06-memory-audit/audit-report.md`
    - Allow override via `--report-path <path>`
    - Log "Loading bge-m3 embedding model..." before audit starts (only if NOT dry-run)
    - Call `auditPensieve({ dryRun, reportPath })`
    - Log summary: "Audit complete: {n} entries reviewed, {m} corrections {dryRun ? 'identified (dry-run)' : 'applied'}"
    - `process.exit(0)` on success

    Run commands:
    - Dry-run: `DATABASE_URL="..." npx tsx src/scripts/audit-pensieve.ts --dry-run`
    - Wet-run: `DATABASE_URL="..." npx tsx src/scripts/audit-pensieve.ts`

    For tests (`src/scripts/__tests__/audit-pensieve.test.ts`): Unit tests on `matchEntryToGroundTruth`, `generateCorrectedContent`, `formatAuditReport`, and AuditResult types. These test pure logic — no DB needed. Mock or skip the `auditPensieve` integration function in unit tests.
