<objective>
Fix the three wellbeing-path race surfaces (RACE-03, RACE-04, RACE-05) in a single plan because they all live in `src/rituals/wellbeing.ts` and share the `findOpenWellbeingRow` ‚Üí `handleTap` / `handleSkip` callback flow.

Purpose:
- RACE-03 closes the rapid-tap completion duplicate-fire surface: under the third-tap, two concurrent callback handlers can both reach `completeSnapshot` and emit `wellbeing_completed` twice, edit the Telegram message twice, and redundantly set `skip_count = 0`. The atomic completion-claim UPDATE makes the side-effect path winner-only.
- RACE-04 closes the data-fidelity violation in the skip path: today's full-object metadata overwrite discards any concurrent partial tap that landed between `findOpenWellbeingRow` and the skip UPDATE. Switching to nested `jsonb_set` merge (matching the tap path) preserves the partial data.
- RACE-05 closes the DST-edge cross-day match in `findOpenWellbeingRow`: the existing `date_trunc('day', fired_at AT TIME ZONE tz) = today` filter can match a prior-day open row when DST arithmetic shifts the local-day boundary. The additional `fired_at >= now() - interval '24 hours'` absolute window rejects stale cross-DST rows.

Output: One production file modified, one test file modified (extends Test 5 + adds two new tests).
</objective>
