# Phase 44 Plan 01: REQUIRE_FIXTURES env-gated hard-fail across 10 milestone-gate test files

## Objective

Replace the silent-skip pattern across the M009 / M010 / M011 milestone-gate test files with an environment-gated hard-fail. When `REQUIRE_FIXTURES=1` is set and a fixture manifest is absent, the affected file emits ONE explicit `[CI-GATE] fixture present` test that throws — turning the current `N skipped` CI green into `N+8 failures` red. Local developers running `bash scripts/test.sh` without the env var see the same skip + regen-hint UX they have today.

Pure test-harness mechanism work. No production code is touched. No new fixtures are produced. The 12 tasks below are all parallel-safe within Wave 1 (they touch 10 distinct test files + scripts/test.sh + REQUIREMENTS.md — no shared mutable state).

## Why Single PLAN.md (Not 12 Plans)

All 12 tasks share one mechanism (the 4-line inline env-gated check) and one acceptance contract (gate-test name + error message format). Splitting would inflate planning + executor context with zero correctness benefit. The "single failing test per fixture" decision (D-03) makes each task ~10 lines of straight-line code addition at a well-defined injection point — no cross-task design tension. A future refactor (e.g., centralizing regen hints per Discretion bullet 2) can be a Phase 47-or-later follow-up.
