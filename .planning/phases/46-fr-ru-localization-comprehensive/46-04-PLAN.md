# Plan 46-04 — L10N-04 daily journal prompt localization

## Objective

Convert `src/rituals/journal.ts:51-58` `PROMPTS` from `readonly string[]` to `Record<Lang, readonly string[]>` (6 prompts per locale, CAP-01 cardinality lock). Wire `fireJournal` to call `getLastUserLanguageFromDb` BEFORE `bot.api.sendMessage` so the prompt text sent to Greg matches his detected locale. Extend the journal tests with per-locale assertions. Closes L10N-04.

## Phase Boundary

In scope:
- `src/rituals/journal.ts:51-58` — convert `PROMPTS` array to `Record<Lang, readonly string[]>`. All three locales have 6 prompts in the same conceptual order (CAP-01 + bag-rotation invariant: same index → same conceptual prompt).
- `src/rituals/journal.ts:306-365` — inside `fireJournal`, add a `getLastUserLanguageFromDb(chatId) → langOf` step BEFORE the `bot.api.sendMessage` call. Use the resolved `lang` to index `PROMPTS[lang][promptIdx]`. Default to `'French'` on null (Greg's primary locale).
- `src/rituals/journal.ts:51` `PROMPT_SET_VERSION` — KEPT at `'v1'`. Bag-rotation indices remain valid because all three locales have the same length (6 prompts, CAP-01 discipline). DO NOT bump version.
- `src/rituals/__tests__/journal.test.ts` — add per-locale prompt selection tests with seeded RNG asserting the FR / RU prompt text is sent when `getLastUserLanguageFromDb` returns the respective locale.
- `src/rituals/__tests__/journal-handler.test.ts` — update mock setup to mock `getLastUserLanguageFromDb` (returns 'French' by default for existing tests; add a new test variant returning 'Russian'). Existing fire-path assertions stay; prompt text is now locale-dependent so test fixtures need a locale value.

Out of scope:
- `chooseNextPromptIndex` itself — pure index-rotation logic, locale-agnostic; not touched.
- `findActivePendingResponse` + `recordJournalResponse` — operate on prompt INDEX or `promptText` (already free-form). The `promptText` stored in `ritual_pending_responses` is now whatever locale text was sent at fire-time (LOCALE-stamped trail — desirable for longitudinal review).
- Schema / migration changes — none. `prompt_bag: number[]` semantics unchanged.
