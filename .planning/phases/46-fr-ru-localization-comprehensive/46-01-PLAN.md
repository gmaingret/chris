# Plan 46-01 — Locale-strings infrastructure + `qualifierFor` canonical (L10N-05)

## Objective

Create the `src/chris/locale/strings.ts` module as the canonical home for cross-file locale registers, ship the consolidated locale-aware `qualifierFor(c, lang)`, and ship the `normalizeForInterrogativeCheck(s)` helper. Remove the duplicate `qualifierFor` from `src/memory/profiles.ts:675-679`. Closes L10N-05 and lands the shared substrate consumed by Plans 46-02 (L10N-01 display-side qualifier) and 46-03 (L10N-03 regex normalization). This plan is the wave-1 foundation; all other plans depend on it.

## Phase Boundary

In scope:
- New module `src/chris/locale/strings.ts` exporting `qualifierFor(c: number, lang: Lang): string` + `LANG_QUALIFIER_BANDS` + `normalizeForInterrogativeCheck(s: string): string`.
- New test file `src/chris/locale/__tests__/strings.test.ts` covering both functions across all three languages plus edge cases.
- Delete the duplicate `qualifierFor` in `src/memory/profiles.ts:675-679`. No other change to profiles.ts (the prompt-side dim labels stay EN per CONTEXT.md D-deferred).

Out of scope:
- Updating `src/bot/handlers/profile.ts` display-side qualifier — that is Plan 46-02 (L10N-01).
- Wiring `normalizeForInterrogativeCheck` into the FR regex consumer — that is Plan 46-03 (L10N-03).
- Touching any other file in `src/chris/` (language.ts, refusal.ts unchanged).
